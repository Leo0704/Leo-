# Skill: 架构审查

当用户要求进行架构审查时，使用此 skill。

## 使用场景

- "分析项目结构"
- "审查代码架构"
- "检查模块边界"
- "评估依赖关系"

## 审查维度

### 1. 项目结构
- 目录组织是否清晰
- 模块划分是否合理
- 文件命名规范

### 2. 依赖管理
- 依赖是否过多
- 是否有循环依赖
- 依赖版本是否稳定

### 3. 代码组织
- 是否遵循项目约定
- 职责是否清晰
- 是否有适当的抽象

### 4. 可扩展性
- 是否容易添加新功能
- 是否有良好的扩展点
- 是否有适当的接口

## 输出格式

```
# 架构审查报告

## 概述
[项目整体]

## 优点
- [优点1评估]
- [优点2]

## 问题
- [问题1] (严重程度: 高/中/低)
- [问题2]

## 建议
1. [建议1]
2. [建议2]

## 总结
[最终建议]
```

## 注意事项

- 给出具体的改进建议
- 区分严重程度
- 提供可操作的步骤
