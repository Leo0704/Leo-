# 快速开始

## 3 步开始

### 1️⃣ 编辑 GOAL.md
描述你想要的**理想状态**：
- 产品愿景
- 成功标准
- 验收条件

### 2️⃣ 发送启动提示

复制以下内容发给 Claude：

```
这是一个目标驱动的开发工作流。

请执行以下步骤：

1. 读取 .workflow/GOAL.md，理解项目的理想状态
2. 读取 .workflow/REALITY.md，了解当前状态
3. 分析差距，判断最需要改进的地方
4. 开始实现第一个核心功能
5. 完成后更新 REALITY.md

目标：让项目尽可能接近 GOAL.md 中描述的理想状态。
```

### 3️⃣ 持续迭代

每次新会话，发送：

```
继续目标驱动开发
```

---

## 文件说明

```
.workflow/
├── GOAL.md      # 理想状态（你想要的）
├── REALITY.md   # 当前状态（实际是）
└── PROGRESS.md  # 进度记录（可选）
```

## 核心理念

**不告诉 AI 做什么，而是告诉 AI 你想要什么。**

AI 会自己：
- 分析差距
- 决定行动
- 执行验证
- 持续迭代
